IRin by Helius

IRin - "IR input". Device (usb dongle) what receive signal from IR remote and send it to virtual com-port via USB (cdc-class). Build on Atmel ARM7 AT91SAM7S64 microcontroller. Use Atmel usb framework.

Main goal

I wanna to build simple device, which receive IR remote signals and send it to OS. No special software, no deprecated com-port, no difficult configuring and bind remote key. I decide use USB communication device class (usb-cdc), because it no need special driver. Just plug usb dongle and read VCP, in linux for example: 
$ cat /dev/ttyACM0
NEC 5916E
NEC 6916E
This asci string is remote key-code (remote has NEC protocol). Just switch through key-codes and make what you need in simple bash script. If you open device in terminal like minicom, you give simple shell for configuring device 
$ minicom -D /dev/ttyACM0
IRin>

Board

clickable

Schematic

Schemaitic is very simple, it contains AT91SAM7S64, 18.342 MHz crystal, 3.3V low drop line regulator, Atmel AT24 eeprom, 2color led for indication, TSSOP1736 IR receiver, few passive components.

Firmware

You can download intell hex file and flash it to chip.

How to flash chip

I use sam_i_am, thanks to Andrew Sterian for cool utility.

Build

You need crosscompiler for build project, i use summon-arm-toolchain scripts for build gcc-arm toolchain. Now you can tune project via Makefile and build it 
$cd src/usb-device-cdc-serial-project/
$make
